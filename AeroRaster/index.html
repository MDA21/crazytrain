    AeroRaster

    AeroRaster - High Performance Software Rasterizer

    Github仓库：AeroRaster

    开发环境：Windows / Visual Studio / C++17 核心依赖：Eigen (数学库), tinyobjloader (模型加载), stb_image (纹理加载)

    项目概述

    这是一个纯 CPU 驱动的软件渲染器，旨在不依赖图形 API（OpenGL/DirectX）的情况下，通过底层优化实现实时渲染管线。主要关注点在于缓存友好性、指令级并行（SIMD）以及多核任务调度的实现。

    技术实现细节

    • 数据布局优化 (Memory Layout)

      • 为了提高缓存命中率，模型数据在加载后被重组为 SoA (Structure of Arrays) 布局，而非传统的 AoS (Array of Structures)。
      • 所有几何数据（顶点、法线、UV）均强制对齐到 32 字节，以适配 AVX2 的 _mm256_load_ps 指令，减少非对齐加载开销。

    • 光栅化核心 (Rasterization Core)

      • 基于 Pineda 算法实现三角形遍历，并使用 AVX2 Intrinsics 进行向量化，单次指令处理 8 个像素的覆盖判断。
      • 引入 BMI1 指令集（_tzcnt_u32 等）实现掩码迭代器，通过位运算直接跳过无效像素块，避免了使用 _mm256_maskstore_ps 带来的微码开销。

    • 多线程任务系统 (Job System)

      • 采用了基于 Tile（分块）的并行渲染架构。
      • 实现了一个基于 std::atomic 的无锁任务队列（Lock-free Queue），支持动态任务分发。
      • 这种设计有效解决了大小核架构（P-Core/E-Core）下的负载不均问题，确保所有核心在渲染阶段保持满载。

    • 管线优化策略

      • 几何阶段：经测试发现，在极短的几何处理任务中，多线程的上下文切换开销大于收益。因此，几何变换与分拣（Binning）阶段采用单线程标量优化（Scalar Optimization），利用编译器别名分析（__restrict）和快速浮点取整指令提升效率。
      • 光栅化阶段：采用 16 线程全并行处理。

    性能测试数据

    • 测试硬件：Intel Core i7-12650H (6P+4E), 16GB RAM
    • 测试场景：Diablo3 角色模型 (3264 顶点, 5022 三角面), Blinn-Phong 光照, 1024x1024 纹理 (Diffuse/Normal/Spec)
    • 分辨率：800x600
    测试项 结果
    平均帧率 ~1300 FPS
    平均帧时 0.75 ms
    几何与分拣耗时 0.075 ms (单线程)
    光栅化耗时 0.68 ms (16 线程 AVX2)
